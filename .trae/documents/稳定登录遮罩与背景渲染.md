## 问题复盘
- 背景是固定的复杂渐变；遮罩层打开后，浏览器在静止阶段可能合并图层或释放优化提示，触发重绘。
- 页面上仍存在 `backdrop-filter`（如导航栏），在遮罩层出现时会与全屏渐变叠加增加重绘概率。

## 下一步修复方案
1. 在遮罩层打开期间禁用导航栏的模糊效果
- 机制：当 `showLoginDialog` 为 true 时，在 `html` 上加 `dialog-open` 类。
- 样式：`.dialog-open .navbar { backdrop-filter: none; }`，避免与渐变背景叠加的昂贵模糊计算。

2. 在遮罩层打开期间冻结背景的动画与过渡
- 样式：`.dialog-open body::before { transition: none; }`，确保背景在遮罩层期间不参与任何过渡。

3. 引入绘制隔离，降低合层与重绘概率
- 样式：`body { isolation: isolate; }` 或对 `body::before` 使用 `contain: paint;`，将背景的绘制与前景分离。

4. 兼容性降级（可选）
- 对支持 `prefers-reduced-transparency` 的环境：在该媒体查询下，使用更轻的背景（弱渐变或纯色），避免任何透明/模糊叠加。
